# SkillSphere — কোড রিভিউ, অপ্টিমাইজেশন নোট ও ইজি ইমপ্রুভমেন্ট

তারিখ: প্রজেক্টের বর্তমান স্টেট অনুযায়ী তৈরি। লক্ষ্য: দ্রুত বোঝা যায় এমন নোট + সহজে ইমপ্লিমেন্ট করা যায় এমন পরামর্শ।

---

## ১. অপ্টিমাইজেশন — এখন কোথায় দাঁড়িয়ে আছে (সংক্ষিপ্ত মূল্যায়ন)

**সামগ্রিকভাবে প্রজেক্টটি মাঝারি থেকে ভালো লেভেলে অপ্টিমাইজড** — বিশেষ করে অ্যাসাইনমেন্ট স্কোপের জন্য যুক্তিসঙ্গত পছন্দ আছে। নিচের টেবিলটি দ্রুত পICTURE দেয়।

| এরিয়া | অবস্থা | নোট |
|--------|--------|-----|
| **ডেটা ফেচিং** | ভালো | `next: { revalidate: 120 }` দিয়ে ISR-স্টাইল ক্যাশিং; তবে একই রিকোয়েস্টে ডুপ্লিকেট কল এড়ানো জায়গা আছে |
| **ইমেজ** | ভালো | `next/image` + `remotePatterns`; কিছু হোস্ট (যেমন `i.ibb.co`) লাগলে কনফিগে যোগ করতে হবে |
| **বান্ডল / ক্লায়েন্ট প্যাকেজ** | মাঝারি | Swiper, Framer Motion, আইকন লাইব্রেরি ঠিক আছে; ব্যবহার না করা ডিপেন্ডেন্সি অপ্টিমাইজেশন হারায় |
| **সার্ভার vs ক্লায়েন্ট** | ভালো | হোম/কোর্স লিস্ট ইত্যাদি সার্ভার কম্পোনেন্ট; অথ ফর্ম ক্লায়েন্ট — ভারসাম্যপূর্ণ |
| **SEO / মেটাডেটা** | ভালো | রুট ও রুট-স্পেসিফিক `metadata`, ডাইনামিক `generateMetadata` কোর্স ডিটেইলে |
| **রুট সিকিউরিটি** | ভালো | `middleware.js` + `proxy.js` দিয়ে প্রটেক্টেড রুট |
| **UX অ্যাক্সেসিবিলিটি** | ভালো | স্কিপ লিঙ্ক, ফোকাস রিঙ`, ফর্ম `aria-*`, লোডিং স্টেট |

**স্কোর (আনুমানিক, অ্যাসাইনমেন্ট প্রেক্ষাপটে):** **৭.৫ / ১০** — প্রডাকশন-গ্রেড পালিশ ও ছোট অপ্টিমাইজেশনের জন্য নিচের ইমপ্রুভমেন্টগুলো।

## ২. ইতিমধ্যে যা ভালোভাবে করা হয়েছে (পজিটিভ নোট)

- কোর্স/হিরো ডেটার জন্য কেন্দ্রীয় `data-fetch.js` ও একই বেস ইউআরএল প্যাটার্ন।
- হোম পেজে একাধিক সেকশনের ডেটা **প্যারালেল** (`Promise.all` বা আলাদা `await` পাশাপাশি যদি থাকে—প্যাটার্ন পরিষ্কার রাখা যায়)।
- `next/font` দিয়ে ফন্ট লোড অপ্টিমাইজড।
- `site-url.js` দিয়ে `metadataBase` ও ডিপ্লয় পরিবেশের সাথে মিল।
- লগইনে `useSearchParams` এড়িয়ে সার্ভার `searchParams` — বিল্ড/প্রি-রেন্ডার স্থিতিশীল।
- কাস্টম `not-found`, লিগ্যাল পেজ, মেটাডেটা টেমপ্লেট।

## ৩. প্রজেক্টকে “বেস্ট” করতে যা দরকার (মিডিয়াম–লং টার্ম)

এগুলো সব “ইজি” নয়, তবে দিকনির্দেশ দেয়:

- **কোর্স ডিটেইল পেজ:** `generateMetadata` ও পেজ উভয়ই `getCourseById` ডাকে → একই রিকোয়েস্টে দুইবার নেটওয়ার্ক/ক্যাশ হিট হতে পারে। রিয়াক্ট `cache()` দিয়ে `getCourseById` র‍্যাপ করা এক লাইনের মতো পরিবর্তনে ডিডupe হয়।
- **হোম পেজ:** `getPopularCourses` ও `getTrendingCourses` ভিতরে আলাদাভাবে `getAllCourses()` → এক রেন্ডারে কোর্স লিস্ট দুইবার টানা। একবার `getAllCourses()` করে সাজিয়ে স্লাইস করা কম রিকোয়েস্ট।
- **ডিপেন্ডেন্সি:** `@heroui/react` ইনস্টল কিন্তু কোডবেসে ব্যবহার নেই → বান্ডল সাইজ/ইনস্টল টাইম বাড়ায়। ইউজ না করলে আনইনস্টল বা ইউজ শুরু করুন।
- **`json-server`:** প্রডাকশন ডিপেন্ডেন্সিতে থাকলে ইমেজ/ডিপ্লয় ভারী হয়। সাধারণত **`devDependencies`** এ রাখা ভালো।
- **`next.config.mjs`:** ইমেজ যেসব ডোমেন থেকে আসে (`images.unsplash.com`, `randomuser.me`, ImgBB ইত্যাদি) সব `remotePatterns` এ আছে কিনা যাচাই করুন।
- **এরর হ্যান্ডলিং:** রুট লেভেল `error.jsx` / গ্লোবাল এরর বাউন্ডারি থাকলে রানটাইম ক্র্যাশ নিয়ন্ত্রণ সহজ হয়।
- **README:** লাইভ লিঙ্ক, env ভ্যারিয়েবল টেবিল, `npm run dev` + `npm run server` ফ্লো বর্ণনা করুন।

## ৪. টপ ১০ ইমপ্রুভমেন্ট — সহজ ইমপ্লিমেন্ট (প্রায়োরিটি অনুযায়ী)

প্রতিটি ছোট কাজ; এক এক করে করলেও লাভ আছে।

| # | ইমপ্রুভমেন্ট | কেন | কীভাবে (হাই লেভেল) |
|---|----------------|-----|---------------------|
| **১** | `getCourseById` এ React `cache()` র‍্যাপ | কোর্স ডিটেইল পেজে মেটাডেটা + পেজ ডাবল ফেচ কমে | `import { cache } from "react"` → `export const getCourseById = cache(async (id) => { ... })` |
| **২** | হোমে একবার `getAllCourses()` | এক রেন্ডারে দুইবার ফুল লিস্ট টানা বন্ধ | `(main)/page.jsx` এ একবার ফেচ করে পপুলার/ট্রেন্ডিং স্লাইস করুন অথবা `data-fetch.js` এ একটা `getCoursesSummary()` হেল্পার |
| **৩** | `@heroui/react` সরানো বা ব্যবহার | অপ্রয়োজনীয় ডিপ ওজন কমে | ব্যবহার নেই → `npm uninstall @heroui/react`; ব্যবহার করতে চাইলে একটি বাটন/মোডাল দিয়ে শুরু করুন |
| **৪** | `json-server` → `devDependencies` | প্রড বিল্ড স্লিম | `package.json` এ সেকশন মুভ + ডক আপডেট |
| **৫** | `next.config.mjs` এ ইমেজ হোস্ট পূর্ণতা | রানটাইম ইমেজ এরর এড়ানো | যেসব CDN ব্যবহার করছেন সব `remotePatterns` এ যোগ |
| **৬** | `README.md` সম্পূর্ণ করুন | সাবমিশন ও ডিপ্লয় সহজ | ফিচার তালিকা, env, লাইভ লিঙ্ক, ব্যবহৃত প্যাকেজ |
| **৭** | `(main)/error.jsx` যোগ | এক রুট ভেঙে গেলে পুরো সাইট নষ্ট না হয় | নেক্সট ডক অনুযায়ী সাধারণ এরর UI |
| **৮** | `npm run lint` প্রি-কমিট / CI | ছোট বাগ আগে ধরা | হুক বা GitHub Actions এক লাইন |
| **৯** | গুগল OAuth (রিকোয়ারমেন্ট অনুযায়ী) | অ্যাসাইনমেন্ট পূর্ণতা | Better Auth ডক + `auth.js` এ প্রভাইডার |
| **১০** | ইনপুট ফিল্ডে সামান্য `prefetch` বন্ধ বা লিঙ্ক `prefetch={false}` যেখানে লাগে | প্রাইভেট রুটে অপ্রয়োজনীয় প্রিফেচ কমাতে | শুধু প্রয়োজনীয় লিঙ্কে—খুব ছোট টিউন |
